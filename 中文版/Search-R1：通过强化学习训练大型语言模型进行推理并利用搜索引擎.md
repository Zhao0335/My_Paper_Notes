# 阅读目标：学习如何使用RL来训练LLM使用工具

# 泛读
## 研究背景
### 存在问题
1. **LLM 不完全具备与搜索引擎交互的能力**：
	LLM没有对使用搜索引擎进行过针对性的学习，训练数据中缺少相关内容。即使通过Prompt进行引导，仍然效果欠佳。
2. **RAG的静态性**：
	使用RAG只能做到根据用户问题，或者通过硬编码分阶段中使用RAG。缺乏主动性和灵活性，难以处理需要多次检索的任务。
3. **微调训练**：
	该方法具有更强的适应性，但由于依赖大规模、高质量的带注释轨迹，且搜索操作存在固有的不可微性，导致基于端到端梯度下降的优化方法无法适用，因此难以有效扩展。
### 结论：
我们应该训练LLM与搜索引擎交互，同时又能够做到端到端的训练 ——> 强化学习
### 合理性：
已被证实，借助强化学习，即使仅在结果奖励上进行训练，模型也能学习到复杂的推理能力，包括自我验证。

### 关键挑战
1. 强化学习框架与稳定性——目前尚不清楚如何将搜索引擎有效整合到大型语言模型的强化学习方法中，同时确保优化的稳定性，特别是在纳入检索到的上下文时。
2. 多轮交错推理与搜索——理想情况下，大型语言模型应能够进行迭代推理和调用搜索引擎，并根据问题的复杂性动态调整检索策略。
3. 奖励设计——为搜索和推理任务设计有效的奖励函数仍然是一个根本性挑战，因为不清楚简单的基于结果的奖励是否足以引导大型语言模型学习有意义且一致的搜索行为。


## 创新点

- **理念创新**：将搜索引擎建模为环境的一部分，生成交错了大语言模型 token 生成与搜索引擎检索的采样轨迹序列；采用了一种简单直接的基于结果的奖励函数，避免了基于过程的奖励所带来的复杂性。
- **技术创新**：提出一种新的强化学习框架SEARCH-R1，支持多轮检索和推理，当被\<search\>和\</search\>标记明确触发时会调用搜索。检索到的内容被包含在\<information>和\</information>标记内，而大语言模型的推理步骤则被包裹在标记中。最终答案使用标记进行格式化，以实现结构化的迭代决策。
- **实验创新**：进行了系统性实验来证明SEARCH-R1的有效性，在相同的实验设置下（例如，相同的检索模型、训练数据和预训练大语言模型），两个大语言模型相对检索增强生成基线分别实现了41%和20%的平均相对提升。
